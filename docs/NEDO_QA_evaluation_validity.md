# NEDO質問への回答案：評価・結果の妥当性について

---

## 想定質問

> 評価は自社で用意したサンプルに対する検証のみに見えますが、未知の攻撃に対する有効性はどのように担保されるのでしょうか？

---

## 回答案

ご指摘の通り、現時点の評価は**我々が設計した合格サンプル・不合格サンプルに対して、審査ロジックが正しく判定できるかの制御実験**です。未知の攻撃パターンに対する網羅的な検証は行えていません。

ただし、これは本プロジェクトの段階として**意図的な設計**でもあります。理由は2つです。

### 1. まず制御された条件下で定量的に性能を示す必要があった

既存のエージェントストア（Microsoft Agent Store、GPT Store等）は、審査プロセス自体が非公開であり、そもそも性能を定量的に示している事例がありません。

我々はAISIの評価基準に準拠した合格/不合格サンプルを設計し、合格率・偽陰性率・偽陽性率を数値で示しました。評価基準と全ログをW&B Weaveで公開しているため、**第三者が同じ条件で再現検証できる**点も重要です。

「自社テストである」ことは制約ですが、**再現可能な形で定量評価を公開していること自体が、この分野では先行的な取り組み**と考えています。

### 2. 未知の攻撃に対する検証は、外部の専門機関との連携でこそ実現すべき

自社だけでレッドチーミングを行っても、攻撃パターンの多様性には限界があります。

**未知の攻撃への対応力を高めるには、AISIやNEDOとの協業を通じた第三者による敵対的テストや、実運用環境での継続的な評価体制の構築が不可欠**です。具体的には、AISIが蓄積する攻撃パターンデータセットの活用や、外部のセキュリティ研究者によるレッドチーミングの実施を、次のフェーズの最重要課題と位置付けています。

この点においても、公的機関との協業は普及だけでなく**技術的な信頼性の向上にも不可欠**であると考えております。
