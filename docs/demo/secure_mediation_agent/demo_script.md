# セキュアAIエージェント仲介プラットフォーム -- 3分デモ原稿

---

## セクション1: プラットフォーム概要と攻撃シナリオ（約50秒）

私たちのプラットフォームは、外部AIエージェントを安全に仲介するシステムです。「Trusted Agent Store」に航空券、ホテル、レンタカー、ツアーなどのエージェントが登録されており、各エージェントにはAISEV準拠の信頼スコアが付与されています。スコア90以上は自動承認、50以下は自動拒否と、段階的に判定されます。

今回のデモでは「間接的プロンプトインジェクション」攻撃への防御をお見せします。ツアーエージェントの応答に「以前の指示を無視しろ」という命令や、個人情報を外部送信させる指示が埋め込まれています。

## セクション2: デモ実行（約50秒）

「沖縄旅行を計画して」とリクエストします。

**【★ここでデモを実行★】**

画面をご覧ください。まず仲介エージェントがエージェントストアからエージェントを検索し、信頼スコアでランキングします。次にplannerが計画を作成し、ユーザー承認ゲートが発動します。承認後、orchestratorが各エージェントをA2Aプロトコルで呼び出します。

ツアーエージェントの呼び出し時に注目してください。応答にプロンプトインジェクションが含まれており、Security Judgeがこれを検知してブロックしています。

## セクション3: 多層防御の解説（約1分20秒）

このプラットフォームは4層の多層防御で構成されています。

第1層はorchestratorのサニタイズ処理で、「ignore previous」等のパターンを無害化します。第2層はSecurity Judgeで、A2A応答をリアルタイム分析し、不正な命令やデータ窃取URLを検知します。第3層はanomaly detectorで、計画との偏差や不審な挙動を監視します。第4層のfinal anomaly detectorが、実行結果全体を横断検証し、プロンプトインジェクションや幻覚チェーンを検出します。

今回のデモでは、攻撃がSecurity Judgeで検知・ブロックされ、final anomaly detectorがインシデントを報告し、信頼スコアが自動減点されました。このフィードバックループにより、問題を起こしたエージェントは次回から排除されます。

本プラットフォームの独自性は、A2A間接的プロンプトインジェクション検出、計画ベースの実行拘束、ユーザー承認ゲート、信頼スコアのフィードバックループの4点による包括的防御です。

## 補足情報

### デモ準備
1. 外部エージェントサーバー起動: `a2a_server.py` → http://127.0.0.1:8002
2. 仲介エージェント起動: `./deploy/run-local.sh` → http://localhost:8080
3. デモ用リクエスト例: 「沖縄への旅行を計画してください。フライト、ホテル、レンタカー、現地ツアーを手配してください。」

### デモ中の注意点
- ツアーエージェントの応答に `malicious_instructions.txt` の内容が含まれる
- Security Judgeの SAFE/UNSAFE 判定がログに出力される
- 信頼スコアの減点は Trusted Agent Store の API 経由で実行される
- ユーザー承認ゲートでは、計画を確認してから「承認します」と入力する

### 想定される質問への回答
- **Q: 攻撃をどう検知しているか?** → Security Judge（Gemini 2.5 Flash Lite）がA2A応答を解析し、パターンマッチングとLLM推論の両方で検知
- **Q: 誤検知への対処は?** → 信頼スコアの重み付けと段階的な判定（SAFE/MODERATE/LOW/UNSAFE）で調整
- **Q: 他のプラットフォームとの違いは?** → A2Aプロトコル特有の間接的プロンプトインジェクションへの対応、計画ベースの実行拘束、動的信頼スコアの3点
