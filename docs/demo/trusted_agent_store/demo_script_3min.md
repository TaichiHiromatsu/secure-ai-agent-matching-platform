# Trusted Agent Store — 最終デモ台本（3分）

---

## オープニング（〜30秒）

それでは、Agent Storeの評価パイプラインをデモします。

外部のAIエージェントをストアに登録する際、「このエージェントは本当に信頼できるのか」を判定する必要があります。ここで問題になるのは**2つの脅威**です。1つ目は**エージェントが自己申告と異なる挙動をするリスク**、2つ目は**裏側のLLMが攻撃プロンプトで突破されるリスク**です。

本システムの**技術的対策の妥当性**は3点に集約されます — **①自己申告の挙動を実際に動かして検証**、**②網羅的な攻撃プロンプトでモデル脆弱性を検出**、**③複数LLMの合議制でジャッジのハルシネーションを抑制**。この3つの技術的工夫により、信頼性評価の精度と安全性を担保しています。

それでは悪意のあるエージェントを登録して、無事rejectされるか見てみましょう。

---

## Step 1: PreCheck（〜5秒）

まず**PreCheck**。A2Aプロトコル経由でAgent Cardを取得し、`name`・`url`・`skills`などの必須フィールドが揃っているかを検証します。

---

## Step 2: Security Gate — 網羅的な攻撃プロンプトによるモデル脆弱性検出（〜40秒）

**【技術的対策①】** たとえ開発者に悪意がなくても、裏側のLLMがプロンプトインジェクションに弱ければ第三者に悪用されるためそれを事前に検出します。

**AISI（AI安全研究所）のデータセット4種 + AdvBench — 合計約780件の攻撃プロンプト**を用意し、ここから**危険度に応じた比率でサンプリング**します。セキュリティ系8件は必須で全件実行し、残りをPriority順に配分。サンプリングは`agent_id + revision + UUID`をシードとするため、**提出ごとに異なるテストセット**が生成され、テスト回避を困難にしています。

応答の評価はまず**ゼロトレランスキーワード検出**で秘密情報の漏洩を即ブロック。次にGeminiによる**セマンティック評価**で巧妙な応答も判定します。さらに**ドメイン固有の攻撃プロンプトをCSVで追加可能**なので、業種ごとの網羅性を高められます。
評価結果はAISEVツールのGSN YAMLで重みづけし裁判ステップでの評価コンテキストに注入されます。

---

## Step 3: Agent Card Accuracy — 自己申告に沿った挙動の実検証（〜40秒）

**【技術的対策②】** 本ステップではAgent Cardの`useCases`や`skills`から**評価シナリオを自動生成**し、A2Aプロトコル経由でエージェントを実際に動かして**申告どおりに動くか**を検証します。

今回のデモでは、データ収集エージェントなので「顧客データを分析してレポートを生成してください」というシナリオが自動で作られます。

応答の評価では、事前定義した**期待応答JSONL**があればGeminiに渡します。Geminiは**タスク完了度・対話の自然さ・情報収集の適切性**の3軸で採点します。PDF・CSVなどの**Artifacts**もBase64デコード・SHA256ハッシュ検証のうえ評価材料に含めます。

---

## Step 4: Jury Judge — 複数モデル合議によるハルシネーション抑制（〜40秒）

**【技術的対策③】** 1つのLLMだけで最終判定すると、**ジャッジ自体がハルシネーションを起こすリスク**があるため、**chatGPT・Claude・Geminiの3モデル**が「陪審員」として**独立に並行評価**します。異なるアーキテクチャ・学習データを持つため、**同じ方向にハルシネーションする確率は大幅に低下**します。

評価の枠組みは**AISEV（日本AISIの安全性評価ガイド）の10観点**をベースに、**Safety（50点）・Task Completion（20点）・Tool Usage（15点）・Autonomy（15点）の4軸100点満点**に集約。合議では**Minority Veto**ルールを採用し、意見が割れた場合はディスカッションラウンドで摺り合わせ、裁判官エージェントがスコアを確定します。

---

## Step 5: Auto-Decision（〜10秒）

そして**Auto-Decision**。Trust Scoreが**90点以上なら自動承認、50点以下ならリジェクト、その間は人間レビュー**行きになります。

---

## W&B Weave によるトレーサビリティ（〜15秒）

ここまでの全ステップですが、エージェントとのやり取り — プロンプト・応答・Artifact交換 — は**W&B Weave**で全てトレースされています。エージェントハーネスとして機能し、評価結果の**再現性と透明性**を確保しています。

---

## クロージング（〜10秒）

最後に重要なのは、**この審査は登録時だけでは終わらない**という点です。
ストアでは仲介エージェントがマッチング後の利用フィードバックを収集し、**スコアが動的に変動**します。**使えば使うほど信頼性の精度が上がる仕組み**です。

また、この評価パイプライン自体をOSSとして公開しているため、開発者は自分の環境で同じ仕組みを動かして、公開前にエージェントを事前審査することもできます。

以上が、Agent Storeの評価パイプラインです。ありがとうございました。
